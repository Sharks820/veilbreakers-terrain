## BUG-60* CONFIRMATION

**Status: CONFIRMED still present at `_terrain_noise.py:1116`.**

```python
1113:            delta_h = new_h - old_h
1114:
1115:            # Sediment capacity based on slope, speed, and water volume
1116:            slope = max(abs(delta_h), min_slope)
```

The `abs(delta_h)` bug is live in the `hydraulic_erosion` function (Python particle simulator). It was NOT fixed in any commit referenced in the git log for this directory. Impact: every call to `_terrain_noise.hydraulic_erosion` (and any caller chain rooting to it) produces physically wrong sediment capacity for uphill-moving droplets.

Note: the *other* erosion implementation in `_terrain_erosion.py:190-207` (which uses `apply_hydraulic_erosion_masks`) does **not** have this bug — it uses `c = max(-h_diff, effective_min_slope)` at line 236, which is geologically correct. The two erosion implementations diverge on this exact point, and `_terrain_world.py` routes through `apply_hydraulic_erosion_masks` (the correct one) for the controller path, so most production paths are unaffected. But direct callers of `_terrain_noise.hydraulic_erosion` (if any) get wrong physics.

Recommend: either patch `_terrain_noise.py:1116` to `slope = max(-delta_h, min_slope)` (matching the newer module) OR deprecate `_terrain_noise.hydraulic_erosion` in favor of `_terrain_erosion.apply_hydraulic_erosion_masks`.
